# BOSS 收藏牛人沟通固定方案

## 定位

这个文档处理"从收藏牛人页与候选人继续沟通"的特殊流程。

收藏牛人与推荐牛人的关键区别：
- 推荐牛人：首次打招呼 → 创建新线程 → 进入聊天
- 收藏牛人：候选人已被收藏但可能没有聊天线程 → "继续沟通"按钮在 iframe 内

## 页面入口

导航到 iframe 实际 URL：`/web/frame/recommend/interaction`

## 前置条件

同 boss-chat 通用前置条件，另加：
1. 候选人已在收藏列表中
2. 已确认收藏列表中确实存在目标候选人

## 固定主路径

1. `/navigate` 到 `/web/frame/recommend/interaction`（顶层页面操作）
2. eval 读取收藏列表，确认目标候选人存在
3. 在当前页面用 `/clickAt` 点击目标候选人的"继续沟通"按钮
4. 如果"继续沟通"按钮点击成功，检查是否跳转到聊天页或新线程已创建
5. 如果候选人在聊天列表中没有线程：
   - 需要先通过推荐页打招呼创建线程
   - 不在聊天列表中反复搜索不存在的线程

## 禁止项

- 不要在 iframe contentDocument 中用 el.click() 点击"继续沟通"按钮（isTrusted=false，Vue 不响应）
- 不要在聊天列表中反复搜索不存在的候选人线程
- 不要一失败就要求用户手动操作

## 成功确认

1. 当前页面是否跳转到聊天页
2. 聊天列表是否出现新线程
3. 线程标题是否包含目标候选人姓名

## 失败处理

如果"继续沟通"按钮点击失败：
1. 先检查按钮是否在 iframe 内（使用 boss-iframe-topology.md 的策略）
2. 如果按钮在 iframe 内且 `/clickAt` 无法操作 → 导航到 iframe URL 再点
3. 如果导航后仍失败 → 进入 `paused_for_favorite_geek_communication`，告诉用户当前收藏候选人沟通路径不可用
